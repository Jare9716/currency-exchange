## ADDED Requirements

### Requirement: API Error Interception and Translation
The system SHALL intercept all API error responses from the backend, translating standardized error codes and English detail messages to Spanish before propagating them to the UI/presentation layer.

#### Scenario: Successful translation of a standardized error code
- **WHEN** the backend API returns an error with the code `"TOKEN_INVALID"` and an English detail message
- **THEN** the HTTP client intercepts the error, resolves the code in the `"api"` translation namespace, and throws an `ApiError` whose `detail` and `message` properties contain `"La sesión ha expirado o el token no es válido. Por favor, inicia sesión de nuevo."`

#### Scenario: Translation of common error detail messages
- **WHEN** the API returns a response containing a common English detail message such as `"Not authenticated"`
- **THEN** the system translates it to `"No autenticado. Por favor, inicia sesión."`

#### Scenario: Translation of validation errors generated by HttpClient
- **WHEN** HttpClient fails to parse response metadata or generates a validation error message locally
- **THEN** the system translates key properties like `"field"` to `"campo"`, `"invalid value"` to `"valor inválido"`, and the fallback message to `"Ocurrió un error desconocido"`.
